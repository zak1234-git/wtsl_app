# 代码阅读路线（新人 0→1）

> 目标：先建立“系统怎么跑起来”的全局图，再下钻“请求怎么被处理”。

## 第 0 步：读入口，确认生命周期
- 文件：`src/wtsl_app/main.c`
- 必看函数：`main`、`signal_handler`、`wteapp_main`
- 你要回答的问题：
  1) 程序如何启动？
  2) 收到 SIGINT/SIGTERM 怎么退出？
  3) 退出时会清理哪些子系统？

## 第 1 步：读核心总控，拿到初始化顺序
- 文件：`src/wtsl_core/wtsl_core.c`
- 必看函数：`wtsl_core_main`、`wtsl_core_release`
- 重点看：`param_default -> config_read -> init_node -> wtsl_logger_init -> wtsl_http_main -> create_splink_state_watcher`
- 学习输出：画出你自己的“启动时序图（文字即可）”。

## 第 2 步：读配置与日志（这是项目可观测性的基础）
- 配置文件：`src/wtsl_core/utils/wtsl_cfg_manager.c`
  - 必看函数：`config_read`、`config_get/config_set`
- 日志系统：`src/wtsl_core/utils/wtsl_log_manager.c`
  - 必看函数：`wtsl_logger_init`、`wtsl_logger_log`、`parse_config_file`
- 你要弄清：
  1) 配置从哪来（`ENV_CONFIG=/home/wt/env_config`）
  2) 日志写到哪（动态文件名 `/home/wt/log/<name>__<date>.log`）
  3) 日志为何有时不输出（`CONFIG_MODE_PRODUCT` 只保留告警/错误）

## 第 3 步：读 HTTP 主循环（外部请求入口）
- 文件：`src/wtsl_core/protocol/http/wtsl_core_httpmain.c`
- 必看函数：`wtsl_http_main`、`request_handler`、`http_get_cmd`、`wtsl_core_http_deal_post_data`
- 关键认知：
  - `request_handler` 做请求体拼装与方法分发（GET/POST/OPTIONS）
  - 最终进入 `wtcoreapi_parse_http_get_cmd/post_cmd`

## 第 4 步：读 REST 路由与节点接口分发
- 文件：`src/wtsl_core/protocol/http/wtsl_core_http_restful_api.c`
- 必看函数：`wtcoreapi_parse_http_get_cmd`、`wtcoreapi_parse_http_post_cmd`、`wtsl_core_get_nodes_functions`、`wtsl_core_post_nodes_functions`
- 关键认知：
  - URL 被拆段后定位到 `nodes` 资源
  - 再通过 `CALL_INTERFACE` 调用节点函数（如 `get_node_basicinfo`、`start_scan` 等）

## 第 5 步：读节点模型与权限系统
- 文件：
  - `src/wtsl_core/slal/node/wtsl_core_node_manager.h`
  - `src/wtsl_core/slal/node/wtsl_core_node_manager.c`
  - `src/wtsl_core/user/user_manager.c`
- 必看点：
  - `NODE_FUNC_LIST`（接口能力总表）
  - `default_interface/global_perm_map`
  - `interface_wrapper`（权限检查 + 本地/远程分发）
  - `check_permission_user`

## 第 6 步：读节点回调实现（业务真正落地处）
- 文件：`src/wtsl_core/slal/node/wtsl_core_node_callback.c`
- 推荐先看：
  - `wtsl_core_get_all_nodes_info`
  - `wtsl_core_get_node_basicinfo`
  - `wtsl_core_set_node_basicinfo`
  - `wtsl_core_start_scan`
  - `wtsl_core_do_connect`
  - `wtsl_core_do_upload` / `wtsl_core_do_upgrade`

## 第 7 步：读底层设备接口（SLB）
- 文件：`src/wtsl_core/slal/slb/wtsl_core_slb_interface.c`
- 必看函数：`param_default`、`init_node`、`set_node`
- 你要弄清：
  - 哪些参数会真正下发到设备
  - 哪些配置会持久化到 `env_config`
  - 网络桥接和驱动加载在什么时机发生

## 第 8 步：读状态机与线程模型
- 文件：
  - `src/wtsl_core/slal/wtsl_core_splink_state.c`
  - `src/wtsl_core/slal/node/wtsl_core_node_service.c`
  - `src/wtsl_core/slal/wtsl_core_splink_mc.c`
- 核心线程：
  1) `wtsl_core_splink_state_watcher`（状态刷新）
  2) `wtsl_core_splink_node_service`（节点服务入口）
  3) `recv_thread/heartbeat_thread`（UDP 心跳收发）
  4) `wtsl_core_splink_gnode_service`（组播同步）

## 第 9 步：读数据解析与 OTA
- 文件：
  - `src/wtsl_core/dataparser/wtsl_core_dataparser.c`
  - `src/wtsl_core/dataparser/wtsl_core_dataparser_ota.c`
- 必看函数：
  - `wtsl_core_parse_json_setnode`
  - `wtsl_core_parse_json_set_node_advinfo`
  - `data_response_to_json`
  - `wtsl_core_parse_json_extract_file_header`
  - `wtsl_core_firmware_upgrade`

## 新手必读函数 Top 20（附理由）
1. `main`：进程生命周期入口。  
2. `signal_handler`：统一退出路径。  
3. `wtsl_core_main`：核心初始化编排器。  
4. `wtsl_core_release`：系统退出回收逻辑。  
5. `config_read`：配置注入全局运行态。  
6. `wtsl_logger_init`：日志系统与上传线程启动。  
7. `wtsl_logger_log`：所有日志输出汇聚点。  
8. `wtsl_http_main`：HTTP 服务启动点。  
9. `request_handler`：HTTP 收包与方法分发核心。  
10. `wtcoreapi_parse_http_get_cmd`：GET 路由分发。  
11. `wtcoreapi_parse_http_post_cmd`：POST 路由分发。  
12. `interface_wrapper`：权限控制与接口调用门面。  
13. `wtsl_core_get_all_nodes_info`：节点列表聚合返回。  
14. `wtsl_core_set_node_basicinfo`：参数修改主路径。  
15. `wtsl_core_start_scan`：扫描业务主路径。  
16. `wtsl_core_do_connect`：入网连接主路径。  
17. `param_default`：默认参数来源。  
18. `init_node`：硬件与网络初始化核心。  
19. `create_splink_state_watcher`：线程体系入口。  
20. `wtsl_core_firmware_upgrade`：升级处理主路径。

## 已确认 / 推断 / 待确认
### 已确认
- 上述 Top20 函数均能在工作区中定位到定义或调用。

### 推断
- 阅读收益最高顺序是“入口→HTTP→节点接口→SLB→状态机→OTA”，因为请求面最容易形成反馈闭环。

### 待确认
- `wtsl_core_dataparser.c` 与 `wtsl_core_http_restful_api.c` 存在疑似异常文本，建议先确认该分支可编译，再做深读。
