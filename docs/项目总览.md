# wtsl_app 项目总览

## 1. 项目目标（面向新同学）
`wtsl_app` 是一个以 **SLB/SLE 无线节点管理** 为核心的 C 项目，提供：
- 设备初始化与组网（本地无线参数、桥接、DHCP、驱动加载）
- HTTP REST 接口（对外管理入口）
- 节点状态维护与节点间同步（UDP 心跳、组播）
- 日志记录与日志上传（文件 + TCP 上传线程）
- OTA 升级（上传固件头解析、包校验、升级执行）

## 2. 技术栈
- 语言与构建：C + Makefile（支持 `arm/arm64/x64`）
- HTTP 服务：`libmicrohttpd`
- JSON：`cJSON`
- 并发：`pthread`
- 系统交互：大量 `system()/popen()` 调用 Linux 命令
- 目标环境：嵌入式 Linux（依赖 `iwpriv/ifconfig/udhcpc/udhcpd` 等工具）

## 3. 目录地图（学习视角）
- `src/wtsl_app/main.c`：进程入口、信号处理、退出清理。
- `src/wtsl_core/wtsl_core.c`：核心初始化总控（配置、节点、日志、HTTP、状态线程）。
- `src/wtsl_core/api/`：核心 API（版本、链路状态采集等）。
- `src/wtsl_core/protocol/http/`：HTTP 服务、REST 路由分发。
- `src/wtsl_core/slal/node/`：节点模型、节点接口、权限包装、回调实现。
- `src/wtsl_core/slal/slb/`：SLB 硬件/驱动/网络参数接口，设备初始化与参数下发。
- `src/wtsl_core/slal/`：节点状态机、组播心跳、SLE 相关线程入口。
- `src/wtsl_core/dataparser/`：JSON 解析、响应封装、OTA 升级流程。
- `src/wtsl_core/utils/`：配置管理、日志管理、日志上传、网络工具。
- `src/wtsl_core/user/`：用户上下文与权限检查。
- `inc/`：外部头文件（如 `wtsl_core_api.h`、第三方头）。
- `lib/`：预置库（按架构区分）。

## 4. 模块关系总图（文字版）
1) `main()` 启动核心：`src/wtsl_app/main.c::main` → `wteapp_main` → `wtsl_core_main`。  
2) `wtsl_core_main` 依次做：默认参数、配置读取、节点初始化、日志初始化、启动 HTTP、启动状态线程。  
   - 证据：`src/wtsl_core/wtsl_core.c::wtsl_core_main`。
3) HTTP 入口处理请求：`request_handler`（收包）→ `wtcoreapi_parse_http_*`（路由）→ `CALL_INTERFACE`（节点接口）→ `wtsl_core_*` 回调（执行业务）。  
   - 证据：`src/wtsl_core/protocol/http/wtsl_core_httpmain.c::request_handler`、`src/wtsl_core/protocol/http/wtsl_core_http_restful_api.c::wtcoreapi_parse_http_get_cmd/wtcoreapi_parse_http_post_cmd`、`src/wtsl_core/slal/node/wtsl_core_node_manager.h::CALL_INTERFACE`。
4) 节点状态线程持续刷新链路与节点信息，节点服务线程通过 UDP 心跳维护节点列表。  
   - 证据：`src/wtsl_core/slal/wtsl_core_splink_state.c::create_splink_state_watcher`、`wtsl_core_splink_state_watcher`、`wtsl_core_splink_node_service`；`src/wtsl_core/slal/node/wtsl_core_node_service.c::recv_thread/heartbeat_thread/parse_data_heartbeat`。

## 5. 关键数据结构（先认名字，再看字段）
- `SPLINK_INFO`：全局运行态（本机节点信息、链路状态、用户数、互斥锁）。
  - 证据：`inc/wtsl_core_api.h::SPLINK_INFO`。
- `WTSLNodeBasicInfo / WTSLNodeAdvInfo / WTSLNodeInfo`：节点基本信息与高级参数。
  - 证据：`src/wtsl_core/slal/node/wtsl_core_node_manager.h`。
- `WTSLNode / WTSLNodeInterface / PermissionMap`：节点对象、函数接口表、权限映射。
  - 证据：`src/wtsl_core/slal/node/wtsl_core_node_manager.h`。
- `log_system_t`：日志系统状态（模式、文件、上传线程配置）。
  - 证据：`src/wtsl_core/include/wtsl_log_manager.h::log_system_t`。

## 6. 已确认 vs 推断
### 已确认
- 程序主入口是 `main`，并处理 `SIGINT/SIGTERM` 时调用 `wtsl_http_main_stop` 与 `wtsl_core_release`。  
  证据：`src/wtsl_app/main.c::main/signal_handler`。
- HTTP 服务默认端口来自配置 `HTTP_PORT`，默认 8080。  
  证据：`src/wtsl_core/protocol/http/wtsl_core_httpmain.c::wtsl_http_main`。
- MQTT 入口函数目前返回 `-1`，未见完整业务实现。  
  证据：`src/wtsl_core/protocol/mqtt/wtsl_core_mqttmain.c::wtsl_mqtt_main`。

### 推断
- 项目当前“主对外控制面”是 HTTP REST，其他协议目录（HTTPS/Modbus/CAN/RS485）可能处于预留或迁移阶段（多个文件为空）。
- `wtsl_core_http_restful_api.c` 有较多接口分支，说明协议面能力增长较快，后续可能继续通过 `CALL_INTERFACE` 扩展。
