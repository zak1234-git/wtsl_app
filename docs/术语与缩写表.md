# 术语与缩写表

## 1. 项目与模块术语
- **wtsl_app**：主程序名（可执行文件名也为 `wtsl_app`）。
- **wtsl_core**：核心库模块，承载初始化、协议处理、节点管理等。
- **SLAL**：项目内的抽象层目录名（`slal/`），包含 node/slb/sle 等子域。

## 2. 网络与节点相关
- **GNode / TNode**：
  - `NODE_TYPE_SLB_G`：Grant 节点
  - `NODE_TYPE_SLB_T`：Terminal 节点
  - 定义见：`src/wtsl_core/slal/node/wtsl_core_node_manager.h`
- **SLE**：另一类节点/连接能力（`NODE_TYPE_SLE_G/T/P`）。
- **SPLINK_INFO**：全局运行态结构，包含本机节点信息、链路状态、用户数、锁。
  - 定义见：`inc/wtsl_core_api.h`

## 3. 协议与接口
- **RESTFUL_API_BASE**：`api`
- **RESTFUL_API_VERSION**：`1`
- **APIRET**：HTTP 路由返回结构 `{status, data}`。
  - 定义见：`src/wtsl_core/protocol/http/wtsl_core_http_restful_api.h`
- **CALL_INTERFACE**：节点接口统一调用宏（会走权限检查包装）。
  - 定义见：`src/wtsl_core/slal/node/wtsl_core_node_manager.h`

## 4. 配置与日志
- **ENV_CONFIG**：`/home/wt/env_config`，运行参数配置文件。
- **DEFAULT_CONFIG_FILE**：`/home/system.conf`，日志模式配置文件。
- **CONFIG_MODE_ALL/PRODUCT**：日志模式，`PRODUCT` 仅保留警告及以上。
- **WTSL_LOG_***：统一日志宏。

## 5. 数据结构速查
- **WTSLNodeBasicInfo**：节点基本参数（name/mac/ip/bw/channel/type...）。
- **WTSLNodeAdvInfo**：高级参数（cell_id、mib、mcs、功率等）。
- **WTSLNodeInfo**：`BasicInfo + ConnInfo`。
- **WTSLNode**：节点对象（ID、地址、活跃状态、接口表、权限映射）。
- **WTSLNodeInterface**：节点能力函数表（由 `NODE_FUNC_LIST` 生成）。
- **PermissionMap**：每个接口的权限要求映射。

## 6. 命名约定（从代码观察）
- `wtsl_core_*`：核心能力函数前缀。
- `slb_*`：底层设备/无线参数接口前缀。
- `config_*`：配置读写前缀。
- `wtsl_logger_*`：日志系统函数前缀。
- `do_* / get_* / set_*`：动作、查询、设置语义。

## 7. 常见英文术语对照
- heartbeat：心跳
- watcher：监视线程
- reload：重载
- upgrade：升级
- upload：上传
- bridge：网桥
- permission：权限

## 8. 已确认 / 推断 / 待确认
### 已确认
- 多数缩写都可以在头文件中定位到明确定义（`wtsl_core_api.h`、`wtsl_core_node_manager.h`、`wtsl_log_manager.h`）。

### 推断
- `SLAL` 更像“项目内部抽象层命名”，不是通用行业缩写。
