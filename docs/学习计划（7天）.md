# 学习计划（7天）

> 目标：7 天内完成“能读懂、能定位、能改小功能、能排障”的上手闭环。

## Day 1：启动链路与目录认知
- 阅读：
  - `src/wtsl_app/main.c`
  - `src/wtsl_core/wtsl_core.c`
  - `Makefile`
- 每日目标：
  1) 画出启动顺序（main → core 初始化 → 子系统）。
  2) 明确构建产物在哪里。
- 动手点：在 `wtsl_core_main` 的关键步骤前后加日志，观察启动顺序。

## Day 2：配置 + 日志
- 阅读：
  - `src/wtsl_core/utils/wtsl_cfg_manager.c`
  - `src/wtsl_core/utils/wtsl_log_manager.c`
  - `src/wtsl_core/utils/wtsl_client_upload.c`
- 每日目标：
  1) 理解 `ENV_CONFIG` 与 `/home/system.conf` 区别。
  2) 理解日志模式切换规则。
- 动手点：添加一个新的配置键，读出来并打印。

## Day 3：HTTP 主流程
- 阅读：
  - `src/wtsl_core/protocol/http/wtsl_core_httpmain.c`
  - `src/wtsl_core/protocol/http/wtsl_core_http_restful_api.c`
- 每日目标：
  1) 写出 GET/POST 请求完整调用链。
  2) 能说明 URL 如何映射到节点函数。
- 动手点：在 `wtcoreapi_parse_http_*` 中打印 URL 分段结果。

## Day 4：节点接口与权限
- 阅读：
  - `src/wtsl_core/slal/node/wtsl_core_node_manager.h`
  - `src/wtsl_core/slal/node/wtsl_core_node_manager.c`
  - `src/wtsl_core/user/user_manager.c`
- 每日目标：
  1) 理解 `NODE_FUNC_LIST` 与 `CALL_INTERFACE`。
  2) 理解权限位掩码检查。
- 动手点：将一个接口权限临时从 `WRITE` 改为 `READ`，观察行为差异。

## Day 5：节点回调 + 数据解析
- 阅读：
  - `src/wtsl_core/slal/node/wtsl_core_node_callback.c`
  - `src/wtsl_core/dataparser/wtsl_core_dataparser.c`
- 每日目标：
  1) 跑通“修改节点参数”链路的每个函数。
  2) 看懂 `set_node + config_set*` 的一致性逻辑。
- 动手点：对一个字段（如 `channel`）做参数合法性校验并打日志。

## Day 6：状态机与节点同步
- 阅读：
  - `src/wtsl_core/slal/wtsl_core_splink_state.c`
  - `src/wtsl_core/slal/node/wtsl_core_node_service.c`
  - `src/wtsl_core/slal/wtsl_core_splink_mc.c`
- 每日目标：
  1) 理解线程模型和共享数据（`global_node_info`）。
  2) 理解 UDP/组播心跳如何更新节点列表。
- 动手点：在 `parse_data_heartbeat` 增加节点变更统计日志。

## Day 7：OTA 升级与风险复盘
- 阅读：
  - `src/wtsl_core/dataparser/wtsl_core_dataparser_ota.c`
  - `src/wtsl_core/dataparser/wtsl_core_dataparser.c`（OTA 相关函数）
- 每日目标：
  1) 理清 upload/upgrade 两阶段。
  2) 列出你认为最关键的 5 个失败点。
- 动手点：模拟 header 解析失败，确认错误响应路径可观测。

## 每日通用建议
1. 每天至少跟 1 条调用链，写成“函数 1→2→3”。
2. 每天至少记录 3 个“我今天确认的事实（带文件+函数）”。
3. 每天至少提出 1 个“待确认问题 + 验证方案”。

## 待确认清单（建议在 7 天内解决）
- `cfg/PreBuild.sh` 缺失是否构建配置问题。
- `wtsl_core_dataparser.c`、`wtsl_core_http_restful_api.c` 的疑似异常文本是否来自冲突未清理。
- 空协议目录（HTTPS/Modbus/CAN/RS485）是否有分支实现。

## 信息可信度
### 已确认
- 每日阅读路径与函数入口均可在当前工作区定位。

### 推断
- 7 天计划按“从入口到协议再到底层”组织，对新人上手效率最高。

### 待确认
